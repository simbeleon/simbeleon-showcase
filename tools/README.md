# Tools & Utilities

Small scripts, templates, and utilities that make day-to-day work faster. Built to solve specific problems, not to be impressive.

---

## Philosophy

A tool earns its place by being used. If it's not solving a real problem, it doesn't get built. If it's been replaced by something better, it gets removed.

Everything here is either:
- **Operational**: something that runs as part of a real workflow
- **Development**: something that supports the build process on active projects
- **Experimental**: something being evaluated — clearly marked as such

---

## Categories

### Process Automation
Scripts and templates that remove repetitive manual steps from operational workflows. Examples: structured report generators, claim intake templates, contract renewal trackers.

### Development Utilities
Tools that support the Idle RPG project or other development work. Architecture validators, file structure checks, smoke test helpers.

### AI Pipeline Components
Prompt templates, output validators, and schema definitions used in agentic pipelines. See the [AI Agents](../ai-agents/) section for context on how these fit into larger workflows.

## On Tooling Choices

Most operational work runs on Excel (advanced formulas, structured tables, pivot reporting) and Navision (ERP). These aren't glamorous choices — they're the right ones for the environment.

For development and AI work: Python, GDScript, Claude Code, Codex.

The goal is always fit-for-purpose, not technology for its own sake.
